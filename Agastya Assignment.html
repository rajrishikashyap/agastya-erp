System architectureWHAT RUNS BEHIND EACH ANSWER
ClientWeb, mobile app, or WhatsApp. The student asks a question in whatever channel the school already uses.
API gatewayAuthentication establishes which student is asking. Rate limiting blocks spam before it costs a single model token.
RetrieverThe question is embedded and matched against a vector database (Qdrant or pgvector), scoped to this school only so one tenant's data never leaks into another's answers.
Context builderRetrieved curriculum chunks are combined with the student's live records via an MCP tool call to the ERP. Fresh data, never a stale copy.
GuardrailsScope check, confidence threshold, and a K-12 safety filter. These run before generation so a bad request is stopped while it is still cheap.
Groq LLMLlama 3 70B via Groq for sub-second generation. SQL handles all structured queries directly; Groq is reserved for genuinely unstructured questions only.
ResponseA grounded answer with a source citation, logged for evaluation so answer quality improves over time.
Every layer maps to a system I have already shipped: auth and access rules from my SaaS learning portal, MCP tool calls from my lead generation machine, and a sub two second LLM pipeline from my production voice agent handling 30+ calls a day.
